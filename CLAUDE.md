# CLAUDE.md

## 全局编码约定

- 文件编码统一使用 `UTF-8`（不带 BOM）。
- 换行符统一使用 `LF`。
- 缩进统一使用 `2` 个空格，不使用 Tab。
- 默认优先使用 ASCII 字符；仅在中文文案、注释或业务内容确有必要时使用 Unicode。
- 保持 TypeScript 严格模式友好：尽量避免 `any`，必要时补充明确类型。

## Vue 组件约定

- 页面级样式写在对应页面组件内，优先使用 `<style scoped>`。
- 全局 `src/style.css` 只保留主题变量、基础重置和全局背景，不放页面专属样式。
- 路由页面文件命名采用 `PascalCase` + `View` 后缀（例如 `BlogView.vue`）。
- 复杂页面优先按“数据结构 -> 计算属性 -> 事件函数”顺序组织 `script setup`。

## 路由与内容约定

- 一级导航节点变更时，需同步检查 `HomeView` 的图谱主节点配置与 `router` 路由定义。
- 内容类页面（如 AI 专题）使用独立 Markdown 文件维护，避免在组件中硬编码大段文本。

## 提交前检查

- 至少执行一次 `pnpm run build-only`，确保可构建。
- 如果改动涉及类型和逻辑，补充执行 `pnpm run type-check`（若仓库已有历史错误，可在提交说明中标注影响范围）。
