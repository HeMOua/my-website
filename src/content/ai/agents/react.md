# ReAct 架构

ReAct（Reason + Act）让 Agent 在执行过程中交替进行“思考”和“行动”，适合需要多轮工具调用的任务。

## 工作机制

1. 读取任务并生成当前思路。
2. 决定下一步动作（调用搜索、代码执行、数据库查询等）。
3. 观察动作结果并更新思路。
4. 循环直到得到可交付结果。

## 适合任务

- 信息检索与分析报告。
- 需要调用多个外部工具的复杂问答。
- 需要边执行边纠偏的任务。

## 风险与治理

- 风险：推理链过长，成本高且可能偏航。
- 治理：设置最大步骤数、工具白名单、失败重试上限。
- 监控：记录每一步 action 与 observation，便于审计与复盘。
