# 检索与生成

在线阶段决定最终体验，重点在于“召回足够全 + 证据足够准 + 生成足够稳”。

## 召回与重排

- 第一阶段召回：向量召回、BM25、规则召回。
- 第二阶段重排：Cross-Encoder 或 LLM reranker。
- 召回融合：按来源置信度与业务权重融合得分。

## Prompt 组装

- 控制上下文窗口：优先高置信、高新鲜度片段。
- 增加引用标识：让答案可追溯到文档与段落。
- 引入策略提示：要求模型在证据不足时明确说明“不确定”。

## 评估与观测

- 离线指标：Recall@K、MRR、nDCG、Answer Faithfulness。
- 在线指标：首 token 延迟、完成延迟、用户满意度、人工复核通过率。
- 反馈闭环：把用户追问、改写、纠错样本回流到评测集与索引优化。

工程实践上，建议把“检索日志 + 生成日志 + 评估日志”统一到同一观测平台，便于快速定位问题。
