# RAG 总览

RAG（Retrieval-Augmented Generation）是把外部知识检索与大模型生成结合的架构。它的关键目标是：在不微调模型参数的情况下，让回答更贴近业务事实，并降低幻觉风险。

## RAG 核心价值

- 将知识更新与模型训练解耦，知识库更新后可快速生效。
- 对企业私有知识更友好，便于权限与审计管理。
- 可以通过可观测指标持续优化，例如召回率、命中率与答案可追溯性。

## 典型链路

1. 采集数据：文档、FAQ、数据库记录、日志等。
2. 预处理：清洗、切分、去重、元数据标注。
3. 向量化与索引：生成 embedding 并写入向量库。
4. 在线检索：根据查询进行召回与重排。
5. 上下文构造：拼接高价值片段给模型。
6. 生成答案：输出带证据的回答。

## 适用场景与边界

- 适用：知识问答、客服助手、内部搜索、研发文档助手。
- 不适用：强事务一致性、毫秒级硬实时决策、纯数值最优化任务。
- 关键边界：如果底层知识质量差，RAG 只能放大问题，不能凭空修复数据。
